Introduction

Git is a popular version control system which was invented by Linus Torvalds in the year 2005. Before git was invented, people, specifically developers faced huge challenges in keeping track of the changes they made to their source codes. So what exactly does git do?, git is like a tracking device implanted into our computers, it can be used to keep track of almost anything we want it to (mostly files like images or text documents).

Git made it easier for developers and programmers to be able to debug source code, as when a certain feature in an app breaks, the developers can be able to see what it is they most recently added to their source code and immediately roll back to the previous working version of their program.

Installing Git

To install git, if using the terminal, type the following commands:


For Linux, type: Keep in mind that this is for ubuntu and debian based systems.

If on the other hand, you are using Linux but specifically arch based, type the following command in your terminal:

Git Basics: init, add, commit

Git basics including initializing a repository to be tracked by git, afterwards, when development and version tracking begins, tracking using the add command and finally commiting the changes using the commit command.

To initialize an empty git repository, type the following command:

                    
                        git init "repo-name"

The above command initializes an empty git repo in the current working directory of your computer.

When you begin development, you will need to stage changes to your source code. To stage changes, use the following command:

                    
                        git add "filename"

Luckily, if you want to stage all the files in the git repo, you can, using the following command:

                    
                        git add .

The period(.) stands for current working directory in linux file systems.

Finally for this section, to now commit changes, which is to put the changes on the final step awaiting the pushing of changes, use the following command.

                    
                        git commit -m "commit message"

The commit message is a short explanation of the changes you made to your source code.

Feel free to read this freeCodeCamp article onHow to Write Better Commit Messages

Git Status and Log

Git status and log are powerful git tools that are used when we want to check whether we might have skipped a useful step or rather debug issues and problems encountered with version control

Git status will give an update on the current tracking situation.
If 2 files have been staged it will give that update, if some havent been staged, it will give that update.
If some files are awaiting to be pushed to the remote github repo, it will provide feedback and also if there is nothing to track, it will also inform us. Pretty cool right?

To be able to use git status command in your working directory, simply type:

                    
                        git status

As easy as that, the status of your git repo will be displayed.

Git log shows the commit history of a repository. It helps trace changes, see who made them and why.

                    
                        git log

One-line summary per commit:

                    
                        git log --oneline

Limit number of commits shown:

                    
                        git log -n 5

View commits of a specific author:

                    
                        git log --author="Your Name"

View changes with each commit:

                    
                        git log -p

These variations make git log a powerful tool for understanding how your codebase has evolved.

Branching

Branching in Git allows you to diverge from the main line of development and work on features or fixes independently. It helps you isolate changes before merging them into the main project.

Create a new branch:

                    
                        git branch feature-name

Switch to a new branch:

                    
                        git checkout feature-name

Create and switch in one command:

                    
                        git checkout -b feature-name

View all Branches

                    
                        git branch
Merging

Merging brings changes from one branch into another. After finishing work on a feature, you typically merge it into main or master.

First, switch to the target branch (e.g., main):

                    
                        git checkout main

Then merge the feature branch into it:

                    
                        git merge feature-name

If conflicts arise, Git will prompt you to resolve them manually before completing the merge.

Cloning & Pulling

Cloning makes a local copy of a remote repository, while pulling keeps your local repo updated with changes from the remote.

Clone a repository:

                    
                        git clone https://github.com/username/repo.git

Pull the latest changes from the remote:

                    
                        git pull origin main

This is essential when working on multiple machines or in a collaborative project.

Push & Pull Workflow

The push and pull workflow keeps your local work in sync with the remote repository. Always pull before pushing to avoid conflicts.

Add files:

                    
                        git add .

Commit changes: 

                    
                        git commit "your-message"

Push to remote:

                    
                        git push origin main

Pull updates from remote:

                    
                        git pull origin main

This cycle helps maintain a clean, updated codebase across collaborators.

Github Basics

GitHub is a hosting service for Git repositories. It provides features like pull requests, issues, wikis, and collaboration tools. Basic GitHub tasks:

Create a new repository via the GitHub interface.

Link your local repo to GitHub:

                    
                        git remote add origin https://github.com/username/repo.git

Push your code:

                    
                        git push -u origin main

Fork a repo to make your own copy and suggest changes via pull requests.

Github enhances version control with project management tools.

Troubleshooting Tips

Common Git problems can be resolved with simple commands:

See what's going on:

                    
                        git status

Undo the last commit (but keep changes staged):

                    
                        git reset --soft HEAD~1

Abort a merge conflict:

                    
                        git merge --abort

Recover lost work 

                    
                        git reflog

Fix line ending issues on Windows/Linux:

                    
                        git config --global core.autocrlf true

When in doubt, use git status, git log, and documentation or tools like GitHub Desktop to visualize history.
